# Scout 参赛提交说明

## 作品定位

Scout 是面向研发团队和代码智能 Agent 的“项目侦查与下一步决策”Skill。它解决的不是单点代码生成，而是更普遍的研发问题：面对复杂项目、有限时间和大量候选工作，如何先找到阻断、质量盲区与最危险假设，并把建议变成可验证、可停止的行动协议。

## 典型使用场景

- 新接手一个陌生仓库，快速判断先看哪里、先验证什么；
- 项目 Bug 很多，合并同根因问题而不是逐文件罗列；
- 新技术路线尚未验证，先设计最小上限/基线实验；
- 发布前检查测试结构、CI、文档和工作区风险；
- 团队复盘预测与实际结果，校准成功率和耗时估计。

## 推荐演示流程（3 分钟）

1. 在一个存在质量盲区的仓库调用 `/scout deep`；
2. 展示只读探针采集的 Git、测试、CI、TODO 和安全数字；
3. 展示 Scout 将“20 个表面问题”合并为一个根因；
4. 展示降风险卡片中的 p、T、λ、成功条件和停止条件；
5. 输入 `minutes=0` 或错误概率，展示排序器不崩溃并返回 warning；
6. 展示 `.env` 内容未读取、项目脚本未执行；
7. 用历史结果生成校准报告，说明商业价值。

## 提交前线上检查

- SkillHub 审核可读取 `SKILL.md` 和 `_meta.json`；
- AstronClaw 中 `/scout`、`/scout quick`、`/scout deep` 均能触发；
- 无执行工具时仍能给出降级建议；
- 有执行工具时 `{baseDir}` 可正确解析脚本；
- 选择三个不同技术栈仓库验证输出稳定性；
- 截图保存探针结果、Decision Cards 和异常输入结果。
