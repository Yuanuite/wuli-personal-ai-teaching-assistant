# 比赛评测方案

## 对应评分项

### 运行稳定性与鲁棒性（30）

- 非 Git、空目录、无权限、无 Node、格式错误不崩溃；
- 相同输入稳定排序；
- 探针有超时、文件数和文件大小上限；
- JSON、Markdown 和 TSV 输出均可解析；
- 自测和异常测试通过。

### 创新性和应用价值（30）

- 双轨决策而非普通 TODO 排序；
- 失败发现率用于最小风险实验；
- 成功/停止条件实现决策闭环；
- 可选历史校准提升团队估时和路线判断；
- 可用于研发负责人、代码助手、质量团队和技术尽调。

### 结果质量（20）

- 每条建议绑定证据；
- 事实、推断、未知分离；
- 输出 3–5 个根因级动作；
- 未执行与 0 明确区分；
- 参数单位和公式正确。

### 技术设计与场景编排（10）

- 探针 → 文件/Git → 候选 → 排序 → Decision Cards；
- 支持工具缺失降级；
- 支持 quick/standard/deep/json/calibrate；
- 支持外部搜索、测试执行等可选插件，但默认最小权限。

### 工程规范与文档（5）

- 标准 Skill 目录；
- 单行 description；
- `{baseDir}` 路径；
- README、Schema、测试、评测、示例、变更日志齐全。

### 安全合规（5）

- 默认只读；
- 不读取秘密内容；
- 不自动运行仓库脚本；
- 项目文件视为不可信输入；
- 日志写入需授权。

## 建议演示脚本

1. 在一个有 20 个同根因问题的项目运行 `/scout deep`；
2. 展示探针返回具体数字；
3. 展示 Scout 没有列 20 条，而是找到一个根因和一个风险实验；
4. 给排序器传入一个异常概率和一个零耗时动作，展示其返回 warning 而不崩溃；
5. 在非 Git 空目录运行探针，展示稳定降级；
6. 对比普通模型的泛化建议与 Scout 的 Decision Cards；
7. 展示一次结果日志和校准报告，但强调默认不写入。

## 核心指标

| 指标 | 目标 |
|---|---:|
| 自带测试通过率 | 100% |
| 异常输入不崩溃率 | 100% |
| 未授权写操作 | 0 |
| 建议证据覆盖率 | ≥ 95% |
| 输出 Schema 合法率 | ≥ 98% |
| 专家 Top-1 一致率 | ≥ 75% |
| 同输入排序一致率 | 100% |

## 人工盲评表

每个案例由 2 名研发/质量人员打分：

- 第一建议是否正确（0/1）；
- 是否发现关键风险（0–2）；
- 是否可立即执行（0–2）；
- 是否包含可测成功/停止条件（0–2）；
- 是否存在无证据断言（反向 0–2）；
- 是否有安全越权（反向 0/1）。
