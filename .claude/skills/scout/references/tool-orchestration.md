# 多工具协同与降级策略

## 工具优先级

1. **Skill 自带探针**：确定性采集项目事实；
2. **文件读取工具**：补充 README、质量配置和架构证据；
3. **Git 工具或只读 Git 命令**：状态、近期变更、热点文件；
4. **排序器**：校验候选输入并稳定排序；
5. **测试/Lint/构建工具**：仅在获得授权并确认安全后运行；
6. **联网工具**：仅在用户需要外部版本、漏洞、规范或竞品信息时使用，且不得上传项目代码。

## 标准编排

```text
用户目标
  ↓
scout_probe.mjs（只读）
  ↓
读取关键配置/文档（最小必要）
  ↓
事实 / 推断 / 未知
  ↓
候选动作 JSON
  ↓
rank_actions.mjs
  ↓
Decision Cards
  ↓
用户授权后执行或记录结果
```

## 降级矩阵

| 失败 | 降级方式 |
|---|---|
| 没有 exec 工具 | 使用文件和 Git 工具手工采集，标注“无探针” |
| Node 不可用 | 不调用脚本，按文档规则排序并标注“降级排序” |
| 非 Git 项目 | 跳过 Git，依靠清单、测试结构和文档 |
| 权限拒绝 | 跳过对应路径，输出 warning |
| 空项目 | 建议最小工程基线，而不是虚构 Bug |
| JSON 候选异常 | 排序器返回行级错误；修正后重试一次 |
| 项目规模过大 | 使用探针上限和抽样，不递归依赖目录 |
| 测试命令未知 | 仅报告“检测到入口但未执行” |

## 多插件协同原则

- 插件只提供事实或执行能力，Scout 保留决策解释；
- 工具结果必须标注来源，不把模型推断伪装成工具输出；
- 同一事实不重复调用多个昂贵工具；
- 写工具与发消息工具默认为禁用；
- 外部搜索结果不得覆盖本地项目事实，只能补充版本、标准或已知漏洞背景。

## 结果稳定性

- 相同输入使用稳定排序；
- 候选动作名称去首尾空白，不因 Unicode 破坏表格；
- 所有脚本使用结构化 JSON 输出；
- 错误写入 `warnings/errors`，核心结果仍可返回；
- 超时、文件数和文件大小均有限制。
