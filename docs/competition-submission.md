# 悟理

> 面向乡村课堂的端侧可信 AI 全流程教学助教平台

**项目口号：让 AI 成为每一位孩子持续生长的土壤。**

## 一句话介绍

教师上传学生错题后，悟理在本地组织 OCR、题干复核、分层解析、按需交互仿真、PDF/学生包交付和错题知识沉淀；教师把关关键结论，学生获得可离线阅读、可检索、可持续复习的个人学习资源。

## 项目定位

当前完整 MVP 聚焦高中物理，覆盖赛道中的“端侧教研搭子”和“课后学习支持”，并把两类能力串成同一条可审计流程。

| 使用阶段 | 悟理提供的能力 |
|---|---|
| 备课与课堂复盘 | 错题 OCR、历史检索、学生版/教师版分层解析、解释图、按需交互仿真 |
| 教师复核 | 原题—题干、题干—答案、答案—仿真三类一致性检查；修改后旧批准自动失效 |
| 课堂讲解 | Markdown/PDF 与可调时间、可缩放、可分层显示的离线 HTML 仿真 |
| 课后支持 | 学生只读站、学生包、错因记录、变式题、复习计划和薄弱点分析 |
| 资源沉淀 | 每道错题形成结构化条目，后续解析与出题可检索复用 |

## 为什么适合乡村课堂

- **弱网可用**：OCR 可在本机完成；PDF、Markdown、HTML 和 ZIP 交付后均可离线使用。
- **保护学生材料**：私有知识库与教师工作台默认只在本机；发布学生站前必须裁剪或遮挡敏感信息并由教师确认。
- **降低教师门槛**：教师可直接说“处理现在新上传的题目”，也可在网页中完成上传、复核、返修与下载。
- **减少重复劳动**：一次复核后的题目继续服务于课堂讲解、学生复习、变式训练和知识薄弱点分析。
- **不要求昂贵软件**：交互仿真是浏览器可直接打开的离线 HTML，不依赖专用仿真平台。

## 核心方案

```text
教师上传图片/PDF
  → 本地 OCR 与图像语义复核
  → 教师确认正式题干
  → 历史检索与分层解析
  → 教师复核答案
  → [教师按需请求交互仿真并复核]
  → Markdown / PDF / 学生包
  → 隐私处理与只读学生站发布
  → 错题检索、复习和变式训练
```

系统不把“生成了内容”等同于“内容已可信”。正式题干、答案和动态仿真分别绑定版本摘要；受保护文件发生变化时，旧复核自动失效，未重新确认的结果不能完成交付。

## 关键创新

### 1. 教师在回路中的可信工作流

AI 负责提取、生成和检查候选结果，教师负责关键事实与教学结论。系统记录复核人、时间和内容摘要，使“谁确认了哪个版本”可以追溯。

### 2. 同题双层解析

- 学生版保留不超过五步的主线、易错点和快速自测，优先采用高中生已知或应知的二级结论，降低认知负担。
- 教师版补充符号、量纲、边界、几何自洽和枚举完备性等审计证据，便于备课复核。

### 3. 答案与仿真共享物理真源

动态题使用同一个 `physics-model.json` 表达区域、物理量、关键事件和轨迹。Markdown 与交互 HTML 不再各写一套答案常量，从源头减少“解析正确、动画错误”的分叉。

### 4. 交互仿真按需生成

所有题目都保留可视化入口，但系统不会为了展示效果强行生成动画。教师明确提出需求后，仿真 Skill 才创建动态过程，并进行静态检查、浏览器运行检查和教师复核。

### 5. 私有教师端与公开学生端隔离

教师端保存原图、教师解析和流程记录；学生端只接收白名单产物。公开题图使用不覆盖原件的副本，公开 ID 不暴露内部条目 ID，静态站也不连接教师端 API。

## 当前可验证成果

截至 2026-07-22，项目仓库中可直接核验：

- 13 道本地真实错题条目当前均处于 `delivered`；
- 4 道题具有共享的结构化物理模型与交互仿真工作流；
- 12 条带适用条件、禁用条件和证据要求的机器可检索二级结论；
- 本地教师工作台、只读学生站、公开题图隐私门禁和离线交付链路；
- 62 项自动化测试，覆盖生命周期、复核门禁、Agent Gateway、模型注册表/逐模型测试、异步作业、学生端发布和页面契约。

这些数字描述当前仓库状态，不代表已经完成学校规模化试点，也不据此宣称学习成绩提升。

## 技术实现

| 层级 | 实现 |
|---|---|
| 生命周期总控 | Python 脚本与 `manage-student-error-library` Skill |
| OCR 与视觉复核 | Apple Vision 本地 OCR；可插拔本地/远程视觉边车；教师人工兜底 |
| 知识与检索 | 本地 JSON、Markdown 和索引脚本 |
| 教师端 | 仅监听 `127.0.0.1` 的本地 Web 工作台与 provider-neutral Agent Gateway |
| 动态可视化 | `build-physics-simulator` Skill、Canvas/SVG、离线 HTML/ZIP |
| 学生端 | 纯静态只读站，可部署到 GitHub Pages |
| 导出 | Markdown、PDF、学生 ZIP 与交付清单 |

主推理模型可以是 Claude Code、Codex、DeepSeek 或其他可调用 Agent。纯文本模型不直接猜图，而是消费已经由视觉边车或教师确认的题干。模型既可来自本机，也可来自经教师授权的远程服务；项目不把联网作为所有流程的必需条件。

## 五分钟演示建议

1. 在教师工作台上传一道含图物理题，展示本地日期文件夹与原图。
2. 对照原图修正并确认题干，触发分层解析，切换学生版和教师版。
3. 修改一句答案，展示旧批准失效，再完成教师复核。
4. 对适合动态讲解的题明确请求仿真，播放并调节时间、缩放和关键事件。
5. 生成 PDF/学生包，裁剪并遮挡原题图，预览只读学生页面。
6. 展示历史检索如何复用错因与二级结论生成后续练习。

## 边界与下一步

当前验证重点是高中物理单学生错题闭环。下一步优先验证真实乡村课堂中的教师时间节省、学生理解度和弱网设备兼容性，再扩展班级管理、Windows/Linux OCR 降级方案，以及数学、化学等学科模型。跨学科是架构方向，不作为当前已完成能力宣传。
